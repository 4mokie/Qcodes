qcodes.Loop¶

class qcodes.Loop(sweep_values, delay=0, station=None, progress_interval=None)[source]¶

Bases: qcodes.utils.metadata.Metadatable

The entry point for creating measurement loops

sweep_values - a SweepValues or compatible object describing what
parameter to set in the loop and over what values
delay - a number of seconds to wait after setting a value before
continuing. 0 (default) means no waiting and no warnings. > 0 means to wait, potentially filling the delay time with monitoring, and give an error if you wait longer than expected.
progress_interval - should progress of the loop every x seconds. Default
is None (no output)

After creating a Loop, you attach action`s to it, making an `ActiveLoop that you can .run(), or you can .run() a Loop directly, in which case it takes the default action`s from the default `Station

actions are a sequence of things to do at each Loop step: they can be Parameter`s to measure, `Task`s to do (any callable that does not yield data), `Wait times, or other `ActiveLoop`s or `Loop`s to nest inside this one.

then(*actions, *, overwrite=False)[source]¶

Attach actions to be performed after the loop completes.

These can only be Task and Wait actions, as they may not generate any data.

returns a new Loop object - the original is untouched

This is more naturally done to an ActiveLoop (ie after .each()) and can also be done there, but it’s allowed at this stage too so that you can define final actions and share them among several `Loop`s that have different loop actions, or attach final actions to a Loop run with default actions.

*actions: Task and Wait objects to execute in order

overwrite: (default False) whether subsequent .then() calls (including
calls in an ActiveLoop after .then() has already been called on the Loop) will add to each other or overwrite the earlier ones.
